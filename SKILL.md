# Skill: Next.js Performance & Build Optimization

## 1. 목적
이 스킬은 애플리케이션의 로딩 속도를 높이고, 라우트 전반의 컴파일 안정성을 확보하기 위한 구체적인 실행 절차를 정의한다.

## 2. 발동 조건 (Trigger)
사용자가 "로딩 속도 개선", "체감 성능", "홈 버튼/내비게이션 수정", "빌드 에러 해결", "TypeScript 타입 수정" 등과 관련된 작업을 요청할 때 이 스킬 프로토콜을 활성화한다.

## 3. 세부 실행 프로토콜

### 단계 1: 데이터 페칭 및 캐싱 최적화
- 데이터베이스 쿼리가 직렬(Serial)로 처리되어 병목이 생기지 않는지 확인하고, 가능한 경우 병렬(Parallel) 구조로 전환한다.
- 페이지 라우트 또는 앱 라우터 특성에 맞춰 최신 Next.js 캐싱 규칙을 적용하고, `revalidate` 관련 레거시 코드가 있다면 즉시 정돈한다.

### 단계 2: 엄격한 타입 체크 및 내비게이션 디버깅
- `any` 타입을 지양하고, 컴파일러가 사전에 오류를 잡을 수 있도록 완벽한 TypeScript 타입을 선언한다.
- 모바일과 데스크톱 레이아웃 간의 컴포넌트 분리 구조를 확인하여, 홈 버튼 등 링크 요소가 유실되거나 이벤트가 씹히는 현상을 해결한다.

### 단계 3: 빌드 시뮬레이션 및 요약 보고
- 모든 수정이 끝나면 컴파일 상태를 확인하기 위해 `npm run build`를 트리거한다.
- 빌드가 완전히 끝나면 성공 여부와 함께 영향을 받은 파일 수 및 추가/삭제된 코드 라인 수(예: `7 files changed +48 -16`)를 정확히 요약하여 사용자에게 보고한다.
